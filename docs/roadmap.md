# Roadmap

## Phase 1: 지금 끝난 것

- 공용 API 클라이언트
- 라이브 목록 조회
- 라이브 상세 조회
- HLS/LL-HLS 선택
- 720p 기준 variant 선택
- 호스트 검증 앱
- 스위치 콘솔 엔트리포인트

## Phase 2: 바로 다음

### 0. 실제 스위치 빌드 고정

먼저 `.nro`가 안정적으로 나오는 환경을 확보한다.

### 1. 영상 재생 백엔드

실질적으로 가장 우선순위가 높다.

후보:

- `libmpv + SDL2` 경로는 이미 빌드되는 상태까지 도달
- 다음은 실기 검증과 안정화
- 필요하면 그 다음 대안으로 외부 플레이어 체인로드 검토

### 2. 실제 UI

현재는 텍스트 콘솔 UI다. 다음 둘 중 하나가 유력하다.

- SDL2 기반 커스텀 UI
- Borealis 기반 네이티브 UI

## Phase 3: 앱다운 기능

- 썸네일 표시
- 설정 저장
- 최근 시청
- 즐겨찾기 또는 그룹

## Phase 4: 후순위

- 채팅 뷰
- VOD
- 검색

## Phase 5: 마지막

- 로그인
- 팔로우/언팔로우
- 인증이 필요한 기능

## 에이전트용 권장 티켓 분해

1. 플레이어 백엔드 조사 및 최소 프로토타입
2. 스위치 UI 프레임워크 선택
3. 썸네일 로딩 파이프라인
4. 로컬 설정 저장 방식
5. 검색/VOD API 확장
